Cost discipline is not about using Cursor less — it's about not wasting money on context and capability you don't need. A few habits keep spend predictable while preserving (often improving) output quality.

## The mental model: you pay for tokens × capability

Roughly, cost scales with how much context you send, how much the model generates, and how capable (expensive) the model is. Every lever below pulls on one of those.

## 1. Default to Auto / Composer

The first-party Auto / Composer pool is tuned to be the cheaper, sensible default for everyday work. Make it your team's standard and only escalate deliberately.

- Set Auto as the default in your [model picker](/learn/feature-tour#model-picker).
- Escalate to a frontier model per-task when you hit a real wall — not preemptively.

## 2. Trim context — the #1 cost leak

The "spray every file at the model and hope" anti-pattern is the single largest source of wasted spend. Each extra file costs tokens *and* dilutes quality.

- Reference the specific files and symbols a task needs — not the whole repo.
- Keep noise out with a strong `.cursorignore` (`node_modules/`, `dist/`, lockfiles, logs). This helps cost **and** [security](/learn/security).
- Break big tasks into focused steps; each step needs less context than the monolith.

<Callout type="tip" title="Precision pays twice">
Tight context lowers cost *and* raises quality. The same discipline that makes output better makes it cheaper. You rarely have to trade one for the other.
</Callout>

## 3. Use Max mode sparingly

Max-style modes pull in more capable models and larger context windows. They're the right tool for genuinely hard reasoning — and an expensive habit for routine work.

- Treat Max as a deliberate, per-task choice.
- A good test: did the cheaper model actually fail here? If you didn't try, don't escalate.

## 4. Mind multi-step agent runs

Autonomous agents iterate — and every iteration spends. A vague task can spiral into many costly steps with little to show.

- Give agents a crisp spec and a clear stopping condition (e.g. "until these tests pass").
- Watch long-running or looping agents; stop and re-scope rather than letting them grind.
- Prefer several small, bounded runs over one open-ended marathon.

<Callout type="warn" title="Vague specs are expensive twice">
An underspecified agent task burns tokens exploring *and* produces a sprawling diff that costs you review time. The cheapest token is the one you didn't need to spend because the spec was clear.
</Callout>

## 5. Set soft spend limits and alerts

Governance beats surprise invoices. Treat spend limits as a control, not an afterthought.

- Configure soft spend limits and alerts so spikes are visible early.
- This doubles as a [security guardrail](/learn/security#enterprise-controls) — a cap limits the blast radius of runaway automation or abuse.

## 6. Monitor per-user and per-model

You can't optimize what you can't see. Review usage by user and by model to find the real cost drivers.

- Spot the handful of workflows or users responsible for most spend and coach or templatize them.
- Watch for expensive-model overuse on tasks the default pool would handle fine.
- Feed findings back into team conventions and `.cursor/rules`.

## The cost checklist

- [ ] Auto / Composer is the team default
- [ ] Context is curated per task, not sprayed
- [ ] Strong `.cursorignore` trims noise
- [ ] Max mode reserved for genuinely hard problems
- [ ] Agent tasks have crisp specs and stopping conditions
- [ ] Soft spend limits + alerts configured
- [ ] Per-user / per-model usage monitored

Most teams cut spend meaningfully just by trimming context and defaulting to Auto — with no loss in quality. Check your config with the **[Config Analyzer](/tools/config-analyzer)** for cost-related tips.
